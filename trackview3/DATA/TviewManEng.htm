1                   INTRODUCTION

1.1      TrackVIEW General

TrackVIEW 3 is the next generation of monorail visualization/management packages. It has the reliability of previous versions of TrackVIEW (100 + installations worldwide since 1994) but includes enhancements Microsoft Visual Basic 6.0 has made available.

1.2      Operating System Requirements

 

         Standard IBM compatible PC.

         Processor - Intel Pentium 4 500 MHz and above.

         Memory 256 MB and above.

         Hard Disk at least 200 MB available.

         Screen Resolution 1024 * 768 minimum. (Large systems will require higher.)

         Screen Colours 16bit (recommended)/256 (emergency use).

         RS232 Communications Ports COM1 and COM2 free. (Master only.)

         Network 10/100 BaseT Ethernet Card.

         Modem.

 

1.3      Standard Features

 

         High-speed communications protocol typical screen update less than 2 seconds.

         15 + Computer Workstations (Full Control).

         If more than one workstation then a slave workstation can backup the databases on the master workstation. If the master workstation fails then the slave can be set as the master. (Move the slave to the master position and change one item in the system configuration.)

         Screen text available in English and most other languages.

         Over 1,000 bags can be displayed.

         99 category codes, 999 customer codes. The ability to add a 25-character name for display/reporting purposes (sometimes shortened to save space). 5 character abbreviated name for displaying in the bags.

         A circle represents bags on the system, with a number in the centre. There are 5 options category or customer code, category or customer name abbreviated or destination.

         Colour allocation to identify category and customer codes on the screen (menu choices no colours, colour = category, colour = customer or colour follows the display option).

         The screen layouts are drawn as close as possible to the actual system layout. Position of the workstation is considered.

         Bag details can be viewed by moving the mouse pointer over the bag. All known information then appears in a pop-up box.

         3 password-protected access levels (Operator, Supervisor and Engineer).

         Full editing of bag information (normally Supervisor level or above).

         Almost unlimited loading/unloading programs (hard disk space is the only limit).

         20 step unloading program for each washing machine. Up to 99 bags can be taken on one program step. 1st, 2nd and 3rd category choices. Option to use Automatic Customer selection.

         Information can be passed to washing machines in many formats (i.e. BCD, Binary, American Binary, Pulsed BCD/Binary and CAN bus.

         Simple visual alarm system. If an operation takes longer than expected then an icon will appear on the screen near to the location of the problem. (Easier for operators to understand than often-meaningless text alarm messages.)

         The ability to start/stop different parts of the system by clicking buttons placed near the item on the display.

         PLC diagnostics. The PLC controls each element of the system using logical sequences, each sequence connected to the next. Engineers can view and adjust the sequences and counters via the TrackVIEW menu.

         Modem support for both the PLC and PCs on the system. The customer must supply a suitable analogue telephone line.

 

1.4      Optional Features

 

TrackVIEW 3 has been designed to be flexible. The following are just some of the possibilities.

         Reporting package see separate description.

         Additional customers and/or categories.

         Alphanumeric displays.

         Tracking of linen on other parts of the system (washing machines, belt conveyors, dryers etc.).

         Semi-automatic re-circulation. (Operator makes selection for operation.)

         Automatic re-circulation. (Options for depth of search.)

         Mixing of Category and/or Customer in bags at Loading / Sorting stations for Report purposes.

         Special or modified loading program.

         Special or modified unloading program.

 

 

2                   MAIN DISPLAY

On power-up the main display will appear. This screen and the menus are always visible to the user.

 

 

 

 

 

2.1      Main Display Functions

2.1.1    Changing the Date

Click the date in the bottom left corner on the main display (the access level must be Supervisor or above).

Click the Date box and enter the new date dd/mm/yy (day/month/year). When the date is correct press the OK button. To leave without any changes press the Cancel button.

2.1.2    Changing the Time

Click the time in the bottom left corner on the main display (the access level must be Supervisor or above).

Click the Time box and enter the current time hh:mm (hour:minutes). When the time is correct press the OK button. To leave without any changes press the Cancel button.

2.1.3    Changing Languages

Changing language couldnt be easier. Click the flag at the bottom of the main display. (Open forms will not change until they are closed and then re-opened.)

Note: We have many language files. Normally two languages are installed, English and the local language, although we can install many more.

 

2.1.4    Changing the Access Level

There are 3 normal access levels, Operator, Supervisor and Engineer. To change the level, click the icon at the bottom right of the main display.

Enter the password for the required access level. (The icon will change to indicate the level.) Once the correct password has been entered the Change button (click to change the password) will function. There is no password for the Operator level.

 

2.2      The Main Menu

The main menu is always visible to the user. However some of the options are access level related.

2.3      View Menu

Under this heading there is at least one choice. All the choices in this menu are related to the system overview. The description should refer to the relevant part of the system.

2.4      Display Menu

This menu has 5 options. These affect what value is displayed in the bags (category code, customer code, destination, category abbreviation or customer abbreviation).

2.5      Program Menu

This menu is where the system programming is done (loading programs, unloading programs and re-circulation).

2.6      Supervisor Menu

This menu has selection for the Supervisor access level.

2.7      End Menu

Clicking End will shut down the PC (Operator level and above).

Note: If the system has more then one PC, the master should only be switched off when no other PCs are switched on.

 

2.8      Window Menu

This is a standard Windows menu for arranging the windows.

2.9      Help Menu

2.9.1    TrackVIEW Manual

Displays this manual on the screen.

2.9.2    Controls Information

Displays the controls system information on the screen (PLC sequences).

 

3                   Loading Programs

Loading programs are the allocation of the customer and category codes to the sorting station bags. Normally the weighing/counting set-points and storage line allocation are automatically taken from the category database.

Refer to Appendix A for any bespoke system functions.

 


Under the weighstations sub-menu there is a selection for each bank of weighstations and, at the bottom, the loading programs menu.

3.1      Stations Bank Menu

Clicking the stations bank menu will bring the loading program for the selected bank into view. When the screen first appears it displays the program that is stored in the PLC.

 

 

3.2      Making a Valid Loading Program

         Enter a customer.

         Enter a category for each station.

         To use the created program press the Send button. Once the PLC receives the information (a box appears for a short time) the operators can continue loading the stations.

Note: Programs cannot be saved from this screen.

3.2.1    Using Saved Program

To use a saved program press the Open button. A list of all saved loading programs then appears. Click the required program then press the OK button. The program should now appear in the Loading Program box.

 

3.3      Making Loading Programs.

To make a loading program that can be stored and used later click the Loading Programs menu option Loading Programs.

Note: There are no station numbers shown on this form. If a system has more than one bank of sorting stations then it will contain enough entry boxes for the largest bank of stations. If, for instance, the system has 1 bank of 8 stations (1-8) and another bank of 4 stations (9-12) then this form will contain 8 sets of entry boxes. The program can be used with either bank of stations.

3.3.1    Open Button

Used to open an existing previously saved program (See Using Saved Program). Once opened it can be edited and saved as the same name or saved with a new name. Changes are only saved if the Save As button is used.

3.3.2    Delete Button

Brings up a similar box to the Open box. Used to remove unwanted programs. The program with the name TVIEW cannot be removed.

3.3.3    New Button

Clears the currently displayed program, leaving the form ready for new entries.

3.3.4    Save as Button

Brings up a similar box to the Open box. Enter a name for the file in the box where the cursor is flashing or select an existing program name from the list.

 

 

3.4      Part-Loads and Purging

The Loading Program form includes a function to release bags that are not full. Depending on the system configuration, to release a bag that has not met the criteria for automatic release (weight or count at predetermined value) the operator can select a station to release. There is also the option to Purge stations (normally the belt would reverse into a Cart).

The above options can also be set automatically. A weight value can be set for release / purging, with options for release and purging. When the Select button is pressed all stations below the weight will be set to purge and all stations above the weight will be set for release. Before anything happens the operator will have the chance to review the options, once confirmed the Release button should be pressed and the operation will begin.

3.5      Progress Boxes

Each station has a 5 stage progress box showing the status of the weight or item count.

Stage Description

1.      Empty, Below 3kg or 0 Items.

2.      Loading. Below Set-point 1.

3.      Near Full, Above Set-point 1.

4.      Full. Above Set-point 2.

5.      Overloaded. Above Set-point 3

 

 

4                   Unloading Programs

Unloading programs control the routing of the bags from the storage lines to the washing machines. Normally each machine has a separate program. It is designed to enable a program to run for long periods without user intervention.

4.1      Basic Unloading Program

 

 

 

4.1.1    Explanation of Basic Functions

4.1.1.1     Step

Each unloading program has 20 steps. The step where the program should start should be selected before sending the program to the PLC.

4.1.1.2     Bags

In this field the number of bags (up to 99) required on a single step should be entered.

4.1.1.3     Line

If required enter a storage line number where the bags should be taken from. Normally a line number or a category would be used, but it may be a requirement to use both together. If a line number is selected then only bags on the entered line will be taken.

4.1.1.4     Category Choice

If required enter a category code (or click the description box for a drop-down list to appear where a selection by name can be made). If a category has been selected then only bags with the same category number will be taken. (Can also be used in conjunction with line number.)

4.1.1.5     Customer Choices

There are 3 customer choices. As with the category enter a customer number (or click the description box for a drop-down list to appear where a selection by name can be made).

Customer Modes

1.      All Customers the program will not use customers.

2.      Manual Selection The required Customer is selected by the operator.

3.      Automatic Selection. The Customer is selected by the PLC.

4.1.1.6     Skip Incomplete Step

Skip Incomplete Step OFF. If the step cannot be completed because there are no matching bags then wait for the bags to be available.

Skip Incomplete Step ON. If the step cannot be completed because there are no matching bags then go onto the next step.

4.1.1.7     Auto Repeat Cycle

Auto Repeat Cycle OFF. Wait on the last step (message shown on the screen) for the operator to change the program.

Auto Repeat Cycle ON. When the last step has completed then go back to step 1.

 

4.1.1.8     Express Line

Any storage line on the system can be allocated as Express. Each Machines can have a different Express line. When a value is set, any bag on the Express line will bag called to the Machine. The Express line takes priority over the normal automatic Unloading program.

4.1.1.9     Manual Unloading

Manual Unloading will interrupt the Automatic program until it has finished. Select the line number required and the number of bags to be taken to the Machine. Once the number of bags has been completed the Automatic program will then continue. To cancel Manual Unloading set the number of bags to 0.

 

4.1.2    Making a New Basic Unloading Program

This example explains how to make a simple program. For the example we will ignore the line number.

For the example we will:

         Set the program to take 6 bags of category 1, 3 bags of category 2, with any customer code and from any storage line.

         Repeat the program until someone stops it.

         Set the program to ensure the order and amount of bags.

         Save the program to the hard disk for future use.

         Open the saved program and send it to the PLC.

         Start it running.

1.      Click the menu [Program Unloading] and select Unloading Programs.

2.      The Unloading Programs form will now appear.

3.      Our first selection is 6 bags of category 1. Therefore click the top Bags entry box and enter 6. As we dont care about the line where the bags come from leave the Line box at 0. Now we need to enter the category 1, click the small box under Category Choice and enter 1. The first Program Step is now complete.

4.      Now repeat 3 but using the 2nd row of boxes and enter 3 bags of category 2.

5.      The program should now look like this:

6.      All the other settings are not done on the saved programs.

7.      We now need to save the program. Press the Save As button and the save box will appear. Enter a name for the file in the box where the cursor is flashing or select an existing program name from the list. Press the OK button and the save is complete. Press the Exit button to close the Unloading Program.

8.      Now we need to use the saved program. Go to the menu [Program Unloading] and select the top machine. (This is normally a name allocated to the project.)

9.      If the program is running (current step highlighted in green) then it will need to be stopped. (All functions are disabled when the program is running.) Press the Stop/Start button. After a few seconds the current step should be highlighted in red indicating the program is stopped.

10.  Now we need to open our saved program. Press the Open button. A program selection box will now appear. (See Using Saved Program example.) Select the program that was previously saved. Press OK. The program should now appear in the boxes.

 

11.  Now we need to set the remaining options. Click the top customer number box and enter 0 (for this program we want bags from any customer).

 

 


12.  Set Auto Repeat Cycle to ON as we want the program to loop around until someone makes changes.

13.  Set the starting step. For this example select the first step.

14.  The program is now complete. To use the program it needs to be sent to the PLC. Press the Send button, a message box will appear for a short period of time. Once the message box has disappeared, the program transfer to the PLC is complete.

15.  To start the program press the Start/Stop button, after a few seconds the highlighted step will change from red to green indicating that the program is running.

16.  This example is now complete.

Note: There are other considerations before bags will move from the storage. Before bags are sent to the washing machine buffer there must be space on the line and the discharger needs to be switched on. When viewing the main layout there are buttons to start and stop each device. Most of these would normally be green running.

 

 


The line status indicator has to be grey for bags to be sent. (See the Engineers section for a full explanation.)

 

 

 

4.2      Advanced Unloading Programs

TrackVIEW 3 unloading programs are Programmer configurable. The Supertrack engineer will set the program options according to the customers needs. In the previous section a basic program was shown. For some customers this may be all that is required (not too much on the screen, making it easy to train operators). However some customers will require the advanced functions.

The advanced program options include:

         2nd category choice.

         3rd category choice.

         Batching (only taking bags in certain quantities).

         Mixing of category choices.

         Automatic Customer Selection.

These options are normally set during the commissioning of the system. However they can be changed at any time (call Supertrack for details).

 

 

 

4.2.1    Explanation of Advanced Functions

 

4.2.1.1     Batch

The Batch is the required number of bags to be selected at any one time. This function is useful when the system has dryers that require more than one load. If, for instance, your dryers require 2 bags and you normally call 8 bags of towels at a time, the program will try to put 8 bags into the machine. However, if at any point one of the bags is not available and the program option Skip Incomplete Steps is switched on, then it might be that only 5 bags were taken (not good for dryer efficiency).

If Batch size was set to 2 then only 4 bags would have been taken to the machine. This is because when the program searches for the bags it must find enough bags to make the batch.

The size of the batch is set up in the category database. However, there is an option in the configuration to allow the size to be changed at the unloading program.

Note: The Standard TrackVIEW II batch limit is 4. However the size of the storage before each machine will limit the batch size. This is because the storage before the machine must contain space for any bags being sent.

4.2.1.2     2nd and 3rd Category Choices

If the 1st category choice is not available then the 2nd or 3rd will be taken. If the Batch option is used then the next section should be read carefully.

 

4.2.1.3     Mixing of Categories

Allow Categories to be Mixed is a option that is Programmer configurable. The Supertrack engineer will set the program options according to the customers needs during commissioning (can easily be changed a later date).

When making a batch, the program reads all the category entries. If the mixing of categories is selected then this will determine how the batch is made up.

Batch search order:

1.      1st Category and Customer.

2.      2nd Category and Customer.

3.      3rd Category and Customer.

If Mix Categories is allowed, then a batch could be made with any of the above mixes. The search always starts at 1 and based on the Mix options will find the 1st bag and then keep adding bags until the batch size is met.

If Mix Categories is not allowed then a batch will be made up of a single category either the 1st, 2nd or 3rd choice. If a Customer is set then the batch will only contain 1 customer.

 

4.2.1.4     Customer Selection Modes

There are 3 modes of operation.

1.      Manual Selection Operator decides which Customer to take from Storage Selection of All Customers also available.

2.      Automatic by Step Gives priority to Machine efficiency - The Unloading program will take bags from the storage starting with the Customer at the top of the list, working down the list until the program step is completed.

3.      Automatic by Program Gives priority to Customers When an Unloading program step cannot be completed using the current Customer, all other steps of the program will be checked before changing to the next Customer in the list. If all of the 20 program steps are searched then the step that was last used will be completed using the next Customer.

 

4.2.1.4.1    Function of Automatic Customer list

When a bag arrives at the storage, if it has a new Customer number then it will be added to the Customer list.

If no bags on the storage lines match the first Customer then it is removed from the Customer list.

Each Unloading Program will use the Customer list independently. Unloading Program 1 could be taking bags for the 1st Customer and Unloading Program 2 could be taking bags for the 4th Customer.

It is possible to add a Customer to the list. If a non-existent Customer is added then it will not cause any problems and will be removed when it reaches the top of the list.

5                   Re-circulation

This section is only relevant if there is a re-circulation line, to take bags from the front of the storage lines to the back.

There are 2 types of re-circulation

         Manual The operator decides the line and amount of bags to take from storage and the line they are put back on.

         Automatic The automatic unloading program uses the re-circulation line for clearing bags that are in front of others that are required for the unloading program step.

 

 

 


5.1      Manual Re-circulation

5.1.1    Ongoing (Bags from Re-circulation Line to Storage)

Select the destination line number and the number of bags required to go. Press the Send button to transfer to the PLC. Bags should then start moving from the re-circulation line to the selected line. To cancel the operation set the number of bags to 0 (press Send).

5.1.2    Offgoing (Bags from Storage to the Re-circulation Line)

Select the line number where bags are to come from and the number of bags required to go. Press the send button to transfer to the PLC. Bags should then start moving from the storage line to the re-circulation. To cancel the operation set the number of bags to 0 (press Send).

Manual offgoing re-circulation should be used carefully as it will stop the automatic unloading program.

 

5.2      Automatic Re-circulation

Automatic re-circulation is very dependent on system layout. Many things affect its operation.

         Distance from storage lines to the re-circulation line.

         The number of washing machines the system feeds.

         The cycle time of the washing machines.

         The number of bags that can be stored before each washing machine.

         The ongoing configuration of the system.

         Number of lifting/lowering devices.

5.2.1    Search Depth

This setting is how far down the storage lines the Unloading Program search can look to find bags. This value needs to be set with care as the greater the number the longer the time it will take for the bags to go to the re-circulation.

Example:

The Search Depth is set to 9. This means the Unloading Program can find a bag on a storage line that is 9 positions from the front. This means to get that bag to the washing machine, the 8 bags in front will be sent to the re-circulation line. (This could take a long time.)

 

6                   Supervisor Menu Functions

The Supervisor menu options are only enabled when the access level is Supervisor or above.

6.1      Edit Storage

This menu is a selection of on or off. If Edit Storage is on then any bags that dont have a value will be shown in grey. This allows any bag on the system to be edited.

 

6.1.1    Editing Bags

Each bag on the system can be edited. It is recommended that the bag movement in the area where editing is required is stopped, because if bags are moving the wrong bag could be changed or a bag could be added.

 

 

When a bag is clicked the Edit form appears.

 

 

 

 


Click the number boxes or the name boxes (drop-down list appears) to edit the values. Once the values are correct, press OK to leave and save the changes or press Cancel to leave without saving the changes.

 

 

 

 

 

6.2      Colours Menu

The Colours menu allows 4 choices of how the bags are shown on the system.

6.2.1    No Colours

Bags are shown all the same colour (normally as the real bags on the system).

6.2.2    Colours = Category

Bag colour is always the colour allocated with the category code.

6.2.3    Colours = Customer

Bag colour is always the colour allocated with the customer code.

6.2.4    Colours = Category or Customer

Bag colour depends on the Display Menu (colour is category if displaying category codes and colour is customer if displaying customers).

6.3      Category Database

Stores all the information connected with the washing category code.

 

The above example category database shows most options. The system configuration dictates what is available in each individual system.

Note: Most of the following functions need to be set at the design stage; they cannot simply be switched on as most require PLC program changes.

 

6.3.1    General

6.3.1.1     Category Description

Enter a category description (up to 25 characters). Try to make the first characters the most meaningful as on some forms the description has to be slightly abbreviated (because of space).

6.3.1.2     Long ID (Abbreviated)

Depending on the system configuration up-to 8 characters can be entered in either numeric, alpha or alphanumeric. This can be displayed in the bag on the screen. See Display Menu.

6.3.1.3     Wash Codes

Used when it is required to pass a different code to the washing machine (normally category code is passed). E.g. normally TrackVIEW has 99 category codes. If the washing machine can only take 15 wash programs then this function could be used.

6.3.1.4     Piece Weight

Used by the reporting program to calculate an item count for a bag. The value should be in grams and an average weight rather than a dry weight.

6.3.1.5     Batch

If the system uses Batch then this is the size for this category. Some categories could go in pairs and some in singles.

6.3.2    Storage line allocations

Each category can have 2 or 4 storage lines allocated.

6.3.2.1     1st

This is the 1st choice of storage line, where the bag will go when leaving the sorting area. If the allocated line has space then bags will always be sent here.

6.3.2.2     2nd

This is the 2nd choice of storage line, where the bag will go when leaving the sorting area. Normally Only used if the 1st choice line is full at the time of leaving the sorting area.

6.3.3    Set-Points

Are used with weighing. The set-points are used for displaying to the operator how much linen is in a bag. Sometimes they are connected to the release of a bag from the sorting area.

6.3.3.1     SP1

This is the value where the bag is nearly full (approaching release).

6.3.3.2     SP2

This is the value where the bag is considered full (at release).

6.3.3.3     SP3

When above this value the bag is overloaded (over release). There are various options of what to do with an overloaded bag depending on the system configuration. See the appendixes.

6.4      Customer Database

Stores all the information connected with the washing category code. This form works exactly the same as the category database. There are only 2 entries to be made, customer name and abbreviation.

 

7                   ENGINEER MENU FUNCTIONS

The Engineer menu options are only enabled when the access level is Engineer.

7.1      PLC Sequences

A PLC sequence controls each system component each sequence interlocks with the next to ensure correct operation of the complete monorail system.

For details of each sequence refer to a section later in this manual - Control System Information.

7.2      PLC Counters

Some parts of the system use counting trips. These are used to determined whether a line is full, has space or a bag is going to the line.

On the above example the counting trips are shown behind the last bag on each line. In reality they are positioned between the front and back trolley (wheels) of the last bag.

How they work:

         When the count is 2 the line is considered clear.

         When a bag is released to the line the count is set to 0 (bag expected).

         When the front trolley (wheel) hits the trip the count is then set to 1 (line full).

         When the back trolley (wheel) hits the trip the count is then set to 2 (line clear).

When a bag is sent to a nearly full line, only the front trolley will make the trip. Therefore the count will stay at 1 indicating the line is full.

Changing counter values is exactly the same as changing sequence step values.

To assist finding which counter needs adjusting, if the PLC Counters form is open, clicking a counter will highlight the counter on the PLC Counters form.

 

7.2.1    Calibration (Weighing)

Once every few months the weighing stations will need to be calibrated to keep the weighing as accurate as possible.

 

How to calibrate a station:

1.      Select the station number you want to calibrate.

2.      Select the Test Weight you are using (about half the normal release weight).

3.      With the station empty and motionless press the Zero button.

4.      Add the Test Weight.

5.      When the Test Weight is motionless press the Gain button.

6.      Check the weight reading is now reading the same as the Test Weight.

7.      Check the station reads 0kg when the Test Weight is removed.

8.      If the above checks are OK then the calibration is complete, if not then it is possible the loadcells may be damaged.

Each PLC sequence has an alarm. Its activation is when something is incorrect for the sequence step or an action takes too long. 9                   GLOSSARY

Discharger = A device on the system where bags are opened (normally into a washing machine).

Form = A screen display

Norm = Normal line allocation (1st choice).

Ongoing = The loading side/part of the system.

Offgoing = The unloading side/part of the system.

Over = Overflow line allocation (2nd choice used if 1st choice is full).

PLC = Programmable Logic Controller (controls the system).

SP = Set-Points. These are normally linked to the weighing. However, they can also be applied to counting systems.

SP1 = Approaching Weight/Count. This is the weight or count value that indicates that the bag is nearly full. If a lamp is fitted to the station it will flash slowly to indicate the station is nearly ready for release.

SP2 = Release Weight/Count Reached. This is the weight or count value that indicates that the bag is full and ready to release to the storage lines. If a lamp is fitted to the station it will be on continuously to indicate the station is ready for release and no more linen should be added.

SP3 = Over Weight/Count. If weight or count is above this value then the bag will not be released (except if an override is fitted). If a lamp is fitted then it will flash fast to warn the operator.

Step = Associated with the PLC sequence. Each component on the system follows a logical step sequence.

10.1Sorting Station Display

 

There is a Sorting Station overview display for each bank of stations. The Long ID is shown for the Category and Customer for each station. The station Category and Customer can be changed from this display.

10.2Counting Stations Display

 

10.2.1           General

The Counting Stations Display is independent of the rest of the system and is used for Reporting only (This allows more Categories for Reporting than the normal 99). It is designed to allow use with a Touch Screen.

10.2.2           Setting the Base Category to a Station

To use the Counting Display correctly a set of Category within the range of 1-99 must first be allocated to each Sorting Stations using the Loading Program. This is the Category that will be used for tracking the bags around the system and then passed to the Washing Machine.

10.2.3           Setting the Category and Customer to be counted

Using the Mouse or Touch Screen click on the required station, The Station number will then be displayed in the Current Station box. To select a new Category or Customer either enter the number or click on the name and a drop-down list with all available options will appear, the drop-down list can be filtered by typing the first letters of the required name.

Once the selection is as required then click Update Selected Station, the selected station count will then be cleared and counting of the new Category / Customer can begin.

 

10.2.4           Changing the Customer for all of the Stations

To change the Customer for all of the Stations select a new customer on any of the Stations and press All This Customer all of the Stations will be set to the new Customer and the Counts will be cleared.
